普通聚合值问题——联合体

我有一个功能

func(union data mydata[])
{
    printf("%llu",(uint64_t)mydata[0]); // Here is the error
}

当我编译此代码时,它给出了以下错误

error: aggregate value used where an integer was expected

您无法访问索引联合数组的字段：mydata [0]是联合数据类型的值,不能转换为uint64_t。

您需要访问正确的union成员：

printf("%" PRIu64, mydata[0].val);

选择uint64_t值.不需要演员.

ip地址使用in_adrr时遇到的问题

struct in_addr {  
    union{
         struct {
                u_char s_b1,s_b2,s_b3,s_b4;
          } S_un_b;
          struct{ 
                u_short s_w1,s_w2;
          } S_un_w;
          u_long S_addr; 
    } S_un;
    #define s_addr  S_un.S_addr
};

in_addr是bit数为32，字节数为4的结构体，内容由联合体引用

struct in_addr addr;//已经有初值

u_long ip;
ip=(u_long)addr；

这样赋值会报，聚合体错误；

struct in_addr addr;//已经有初值

u_long ip;
ip=addr.s_addr;

这样就不会报错
